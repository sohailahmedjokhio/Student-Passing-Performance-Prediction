Project Information

Learn about the Student Performance Prediction system, its methodology, and performance.

Project Overview

The Student Performance Prediction project aims to predict whether a student will pass or fail a course based on various personal, academic, and lifestyle factors. Using a Support Vector Machine (SVM) with a linear kernel, the system provides a probability score and categorizes the likelihood of passing into Low (<40%), Medium (40-70%), or High (>70%).

Built with Flask, the web interface allows users to input 30 features about a student and receive real-time predictions. The project integrates MySQL for data storage and uses Python libraries like scikit-learn, pandas, and matplotlib for modeling and visualization.

Objective: Assist educators and students in identifying at-risk students early, enabling targeted interventions to improve academic outcomes.

Dataset Description

The project uses the Student Performance Dataset from the UCI Machine Learning Repository. It contains data on 395 students from two Portuguese schools, collected during the 2005-2006 school year.

Key Features (30):

  • Personal: school, sex, age, address, family size, parent's cohabitation status, mother's/father's education, mother's/father's job, reason for choosing school, guardian.
  • Academic: travel time, study time, past failures, extra educational support, family support, paid classes, activities, nursery school, desire for higher education, internet access.
  • Lifestyle: romantic relationship, family relationship quality, free time, going out, workday/weekend alcohol consumption, health, absences.
  • Target: passing percentage (eg. 55%), passing category (eg. medium) .

Dataset Size: 395 rows, 31 columns (30 features + target).

Methodology

Data Preprocessing

  • Categorical Encoding: Converted categorical features (e.g., school, sex) to numerical values using mapping dictionaries.
  • Feature Scaling: Normalized features with large ranges (e.g., absences) using mean normalization and small ranges (e.g., studytime) using min-max scaling.
  • Data Splitting: Split dataset into training (60%), validation (20%), and test (20%) sets with a random state of 388628375.

Model Training

  • Algorithm: Support Vector Machine (SVM) with a linear kernel.
  • Hyperparameter Tuning: Optimized the regularization parameter C using validation error minimization over values [0.0001, 0.001, 0.01, 0.05, 0.1, 4, 10, 40, 100].
  • Training Time: Approximately 74ms on the training set.

Prediction

  • Probability Output: Computes the probability of passing using SVM's predict_proba method.
  • Category Assignment: Low (<40%), Medium (40-70%), High (>70%).
  • Web Interface: Flask-based form collects 30 features, processes them, and displays predictions.

Model Details

The model is a Support Vector Machine (SVM) with a linear kernel, chosen for its effectiveness in binary classification tasks with moderate-sized datasets. The linear kernel ensures interpretability and computational efficiency.

  • Input Features: 30 features (personal, academic, lifestyle).
  • Output: Probability of passing (0-100%) and category (Low, Medium, High).
  • Regularization Parameter (C): Optimized via cross-validation.
  • Training Data: 237 samples (60% of 395).
  • Validation Data: 79 samples (20%).
  • Test Data: 79 samples (20%).

The model is saved as svm_model.joblib and loaded for predictions, ensuring fast inference (~1ms per prediction).
